# 🎓 GradeUp - Plateforme de Gestion Scolaire Intelligente

**GradeUp** est une plateforme web moderne et complète de gestion scolaire conçue pour faciliter la communication et l'organisation entre administrateurs, enseignants, élèves et parents.

**Version:** 2.0  
**Créé par:** Axions Labs  
---

## ✨ Vue d'ensemble

GradeUp offre une solution complète pour gérer tous les aspects d'une école :
- 👥 **4 rôles utilisateurs** distincts avec permissions spécifiques
- 📊 **Tableaux de bord personnalisés** pour chaque profil
- 🔐 **Système d'authentification sécurisé** avec codes d'invitation
- 💬 **Communication intégrée** entre tous les utilisateurs
- 🌙 **Mode sombre** pour un confort visuel optimal
- 🤖 **Assistant IA intégré** pour des conseils et suggestions
- 📱 **Interface responsive** compatible tous les appareils
- 🎨 **Design moderne** avec thème bleu premium

## 📋 Architecture de la Base de Données

### Models Principaux

**School**
- Informations de l'école
- Code d'invitation unique
- Devise configurée

**User**
- 4 rôles: Admin, Enseignant, Élève, Parent
- Relations parent-enfant
- Code parent unique par élève
- État actif/inactif

**SchoolClass**
- Niveaux: Maternelle, Primaire, Secondaire
- Frais de scolarité configurables
- Lien à l'école

**Course** (Cours)
- Créés par les enseignants
- Liés à une classe
- Avec description détaillée

**Lesson** (Leçons)
- Contenu pédagogique
- Fichiers attachés
- Éditeur MDX

**Grade** (Notes)
- Par étudiant, par cours, par trimestre
- Score et note max
- Commentaires

**Homework** (Devoirs)
- Titre et description
- Date limite
- Lié à un cours

**Attendance** (Présence)
- Statut: Présent, Absent, Retard
- Raison de l'absence
- Quotidienne

**Payment** (Paiement)
- Statut: Payé, En attente, En retard
- Montant et méthode
- Par mois

**Message**
- Messages directs entre utilisateurs
- Statut lu/non lu
- Horodatage

**Notification**
- Notifications à l'école entière
- Marquer comme lue
- Archivable

## 🚀 Installation et Démarrage

### Prérequis
- Node.js 18+
- npm ou yarn ou bun
- Navigateur moderne

### Installation
```bash
# Cloner le projet
git clone <repository>
cd FullAZ

# Installer les dépendances
npm install
# ou
yarn install
# ou
bun install

# Configurer la base de données
npm run db:push

# Générer le client Prisma
npm run db:generate
```

### Démarrage

**Mode développement**
```bash
npm run dev
# L'app sera disponible à http://localhost:3000
```

**Mode production**
```bash
npm run build
npm start
```

### Commandes disponibles
```bash
npm run dev          # Démarrer en mode développement
npm run build        # Compiler pour production
npm start            # Démarrer le serveur de production
npm run lint         # Vérifier le code
npm run db:push      # Synchroniser la base de données
npm run db:migrate   # Exécuter les migrations
npm run db:reset     # Réinitialiser la base de données
npm run db:generate  # Générer le client Prisma
```

## 📊 API Routes (23Total)

### Authentification
- POST `/api/auth/register` - Créer un compte
- POST `/api/auth/login` - Se connecter
- GET `/api/auth/me` - Obtenir l'utilisateur courant

### Utilisateurs
- GET `/api/users` - Liste des utilisateurs
- GET `/api/users/[id]` - Obtenir un utilisateur
- PUT `/api/users/[id]` - Modifier un utilisateur
- DELETE `/api/users/[id]` - Supprimer un utilisateur

### Classes
- GET `/api/classes` - Liste des classes
- POST `/api/classes` - Créer une classe
- PUT `/api/classes/[id]` - Modifier une classe
- DELETE `/api/classes/[id]` - Supprimer une classe

### Cours
- GET `/api/courses` - Liste des cours
- POST `/api/courses` - Créer un cours
- PUT `/api/courses/[id]` - Modifier
- DELETE `/api/courses/[id]` - Supprimer

### Leçons, Notes, Devoirs, Présences
- Routes CRUD complètes pour chaque entité

### Paiements
- GET `/api/payments` - Liste des paiements
- POST `/api/payments` - Créer un paiement
- PUT `/api/payments/[id]` - Modifier un paiement

### Messages
- GET `/api/messages` - Historique des messages
- POST `/api/messages` - Envoyer un message

### Notifications
- GET `/api/notifications` - Obtenir les notifications
- POST `/api/notifications` - Envoyer une notification
- PUT `/api/notifications/[id]` - Marquer comme lue

### Analytics
- GET `/api/stats` - Statistiques de l'école
